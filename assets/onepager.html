Why AEGIS

🪶 Tiny, auditable core

Full-platform capability in ~11k lines across 81 modules — small enough to read and trust end to end. None of the sprawl.

🔌 Truly model-agnostic

29 provider presets — Claude, GPT, Gemini, Llama, DeepSeek, Qwen, Grok, local Ollama — behind one interface, with API-key and OAuth, fallback chains, and per-prompt routing.

🧠 It actually learns

A real closed loop: reviews finished sessions, extracts memory + skills (secret-redacted), promotes them on your approval. Optional background review + cross-session recall.

🛡️ Safe by default

Permission cascade with a hardline blocklist (refuses rm -rf / even in yolo), pre-exec scanning, fail-closed sandboxes, and prompt-injection-aware tool results.

📡 Everywhere you are

One agent serving CLI, Telegram, Discord, Slack, Signal, Matrix, Email, webhooks, and ntfy — with voice transcription, native file attachments, and a durable delivery queue.

🖥️ Do everything in the browser

A rich dashboard: live activity feed, chat, a kanban board, scheduled tasks, config, sessions, memory, skills — no terminal required.

Batteries included

Tools — files, shell, web, browser automation, code exec, LSP, GitHub, image gen, voice, OSV vulnerability audit.
MCP — connect external tool servers and expose your own.
Gateway — proactive messaging, per-channel formatting, mid-run "stop", in-place status.
Scheduling — recurring + one-shot cron ("remind me at 5pm") delivered to any channel.
OpenAI-compatible API — aegis serve exposes your agent at /v1/chat/completions.
Editor integration — ACP with permission prompts and filesystem delegation.
Trajectory export — jsonl · openai-finetune · sharegpt.
Cost analytics — cache-aware spend, per model.
